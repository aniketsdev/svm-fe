## Common scripts

```bash
bun dev          # start the Vite dev server
bun run build    # type-check + production build
bun run test     # run the Vitest suite once
bun run test:watch  # watch mode
bun run lint     # ESLint (bans @mui/* and @emotion/* imports)
bun run sdk:gen  # regenerate the type-safe API SDK from svm-be's OpenAPI
```

## Generating the API SDK

The TypeScript SDK under `src/sdk/` is generated by [Orval](https://orval.dev/) from
the backend's OpenAPI document. Regenerate after any backend contract change:

```bash
# Requires svm-be running on http://localhost:8000
bun run sdk:gen
```

Configure the source URL via `OPENAPI_URL` in `.env.local` (defaults to
`http://localhost:8000/api/v1/openapi.json`). The wrapper at
`scripts/sdk-gen.sh` pre-checks the URL and refuses to clean `src/sdk/` when
the backend is unreachable, so a failed regen leaves the previously generated
SDK in place.

Full walkthrough, decisions, and troubleshooting:
[`specs/004-frontend-orval-sdk/quickstart.md`](../specs/004-frontend-orval-sdk/quickstart.md).

## Authentication model

The app uses cookie-based authentication backed by `svm-be`:

- **`__Host-access`** (`HttpOnly`, `Secure`) — short-lived (15 min) JWT
  carrying user id, email, role, session id.
- **`__Host-refresh`** (`HttpOnly`, `Secure`, `Path=/api/v1/auth/refresh`) —
  opaque 32-byte token redeemed by `POST /auth/refresh` to rotate sessions.
- **`csrf_token`** (`Secure`, NOT `HttpOnly`) — read by JS for the
  double-submit pattern. Every non-`GET` request must echo it as
  `X-CSRF-Token`.

The FE never stores tokens in `localStorage` / `sessionStorage`. Auth state
flows entirely through cookies + a TanStack Query bootstrap of
`GET /auth/me`. See `src/features/auth/context/AuthContext.tsx` for the
mounted-on-startup bootstrap and `src/api/client.ts` for the CSRF
interceptor (every non-safe method auto-injects the header).

## Open the component library

```bash
bun dev
# then visit:
http://localhost:5173/#__component-library
```

The route only exists when `import.meta.env.DEV` is true; production builds
never expose it.
